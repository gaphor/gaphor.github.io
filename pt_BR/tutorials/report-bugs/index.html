Como relatar um bug no Gaphor

Se você já escreveu uma linha de código, saberá que todo software tem bugs. Este também será o caso com Gaphor. Se você acha que encontrou um bug, basta clicar no link abaixo (você precisará ter uma conta no GitHub) e, em seguida, clique no botão verde “New Issue” encontrado no lado direito superior da página:

Relate um bug no Gaphor

Quando você criar um novo relatório de problema, será dado um número e os desenvolvedores voluntários, que escrevem e mantêm o código, serão informados por e-mail. Eles podem anotar perguntas para o seu problema, se as coisas não são claras. Eles podem imediatamente fechá-lo e marcá-lo como “duplicado” (alguém já relatou o bug – e eles vão referenciar o problema original). Às vezes eles vão fechar o bug e dizer “não corrigir”, porque eles discordam que é um bug ou é muito trivial para investir a qualquer momento.

Existem geralmente dois tipos de bugs:

  • Técnico: o código tem um erro nele que faz com que o programa quebre.
  • Comportamental: o código funciona corretamente, mas está fazendo a coisa errada.

Em ambos os casos, para sermos capazes de corrigir o bug, precisamos tantas informações quanto possível. Quando você cria um novo relatório de problema, você deve tentar incluir, quando relevante, os seguintes tipos de informações:

  • O que você estava tentando fazer,
  • Que passos você tomou para fazer isso acontecer
  • O que você esperava que acontecesse,
  • O que realmente aconteceu,
  • Por que essa diferença é problemática (pode não ser um bug!),
  • Detalhes técnicos como a versão de Gaphor que você está usando, a versão do seu sistema operacional e outros aspectos do contexto em que Gaphor estava executando.

Use linguagem simples e simples para descrever o problema e, se útil, divida-o em etapas para que os desenvolvedores possam facilmente recriar o problema. Por favor, não assuma que vamos entender o que você estava tentando alcançar – honestamente, é melhor se você tentar imaginar que nós (os desenvolvedores) somos um monte de jovens de 5 anos inteligentes. Tente explicar tudo sobre o problema e não assuma que sabemos o que você quer dizer. Não nos importamos!

Se você gostaria de se envolver mais no desenvolvimento da Gaphor, gostaríamos de recebê-lo através do site do desenvolvedor Gaphor.
