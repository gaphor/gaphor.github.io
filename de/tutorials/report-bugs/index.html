Fehler in Gaphor melden

Wenn Sie jemals eine Zeile Code geschrieben haben, wissen Sie, dass jede Software Fehler beinhaltet. Das wird auch bei Gaphor der Fall sein. Wenn Sie glauben einen Fehler gefunden zu haben, klicken Sie einfach auf den unten stehenden Link (Sie müssen ein Konto bei GitHub haben) und dann auf die grüne Schaltfläche „Neues Problem“ auf der Seite oben rechts:

Ein Problem im Gaphor melden

Wenn du ein neues Problem erstellst, erhält es eine Nummer und die freiwilligen Entwickler, welche den Code schreiben und unterhalten, werden per E-Mail informiert. Sie können Fragen zu deinem Problem stellen, wenn etwas unklar ist. Sie können es sofort schliessen und als „duplicate”“(jemand anderes hat das Problem schon gemeldet – und sie werden das ursprüngliche Problem referenzieren). Manchmal werden sie das Problem schliessen und sagen „won’t fix“, weil sie widersprechen, dass es ein Problem ist, oder weil es zu unbedeutend ist, um Zeit zu investieren.

Im Allgemeinen gibt es zwei Arten von Fehlern:

  • Technisch: Der Code enthält einen Fehler, der dazu führt, dass das Programm abstürzt.
  • Verhalten: Der Code funktioniert korrekt, macht aber das Falsche.

In beiden Fällen benötigen wir so viele Informationen wie möglich, damit wir den Fehler beheben können. Wenn Sie ein neues Problem melden, geben Sie nach Möglichkeit folgende Arten von Informationen ein:

  • Was haben Sie versucht zu tun,
  • Welche Schritte haben Sie unternommen die dazu führten,
  • Was sollte passieren,
  • Was tatsächlich passiert ist,
  • Warum ist dieser Unterschied problematisch (es ist möglicherweise kein Fehler!),
  • Technische Details wie die von Ihnen verwendete Gaphor-Version, Ihre Betriebssystemversion und andere Faktoren in dem Gaphor ausgeführt wurde.

Beschreiben Sie das Problem in einer einfacher Sprache und unterteilen Sie es am besten in einzelne Schritten, damit Entwickler das Problem problemlos reproduzieren können. Bitte gehen Sie nicht davon aus, dass wir verstehen was Sie erreichen wollten – am Besten stellen Sie sich vor, dass wir (die Entwickler) ein Haufen schlauer Fünfjähriger sind. Versuchen Sie, alles über das Problem zu erklären und gehen Sie nicht davon aus, dass wir wissen was Sie meinen. Je mehr Informationen wir erhalten umso besser!

Wenn Sie sich stärker an der Entwicklung von Gaphor beteiligen möchten, nutzen Sie die Webseite für Gaphor-Entwickler.
